# Security Policy (Art)

## Репорт уязвимостей
Пожалуйста, **не** публикуйте уязвимости в Issues/Discussion.

До публичного релиза: сообщайте владельцу репозитория приватно.  
После публичного релиза: используйте GitHub “Private vulnerability reporting” (будет включено в настройках репозитория).

## SLA
- Triage: до 72 часов
- Исправление: по severity и согласованному плану релиза

## Что запрещено публиковать
- ключи/токены/пароли
- приватные дампы данных, содержащие PII
- полноценные эксплойты до согласования патча
