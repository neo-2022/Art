# API versioning

- API версия фиксирована в пути `/api/v1/...`.
- Для схем используется `schema_version`; для v1 значение фиксировано как `1.0` (где поле присутствует).
- Изменения в `v1` допускаются только backward compatible.
- Любой breaking change оформляется только через новую версию пути и схем (`/api/v2/...`, `docs/schemas/v2/*`) без правки контрактов v1.
