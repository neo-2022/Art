# Runbook: documentation_drift_detected

## Что это означает
Важный документ изменился, а зависимые документы или дерево документации не были синхронизированы.

## Что проверить
1. Какой документ изменился.
2. На сколько строк он изменился.
3. Какие зависимые узлы дерева затронуты.
4. Не влияет ли он на корневой `README.md`.

## Что делать
1. Пересчитать documentation tree.
2. Обновить зависимые документы.
3. Повторить docs/root sync checks.

## Проверка восстановления
- дерево документации актуально;
- зависимые документы согласованы;
- CI docs gates снова зелёные.
