# Runbook: action_safety_guard_blocked

## Что это означает
Система заблокировала выполнение опасного действия, потому что не увидела достаточного semantic safety proof.

## Что проверить
1. Какое действие пытались выполнить.
2. Был ли preflight.
3. Есть ли bounded-regret / no-regret certificate.
4. Есть ли policy exception и кто её выдал.

## Что делать
1. Не обходить guard вручную без явной policy exception.
2. Собрать missing evidence.
3. Повторить preflight.
4. Если действие всё ещё нужно, оформить исключение с audit trail.

## Проверка восстановления
- повторный preflight проходит;
- действие либо безопасно выполнено, либо отменено с документированным основанием.
